My group and I built this site for a staff member at UC Berkeley who wanted a way to track other faculty memeber's interest in various project he was working on around campus. The biggest technical challenge was syncing with the Google Calendar, LinkedIn, and Berkeley API's to retrieve data and then itegrating that data into the site. Assassins Mobile App
Role: Developer, Designer Tools: Android/Toq Watch (Java, XML), FramerJS, Adobe CC
<
>
Assassins is a social game used as an ice breaker in which a facilitator runs/organizes the game and 2 or more players participate. Each player is given a target who they are supposed to "assassinate" in a method determined by the facilitator. An example would be touching your target with a spoon. I worked in a team of 5 to develop this app for Android and the Qualcom Toq Watch. Our main goal was to make the facilitator's job easier while preserving the main components of the game, namely its social component. Scrapbooking Site
Role: Developer, Designer Tools: HTML, CSS, Javascript
<
>
Scrapi is a site I made to work on my front-end and design skills. It is a scrapbooking site that allows people to collect their photos online and make scrapbooks from those collections. I made 3 views for the site: the homepage, a page to view all collections, and a page to view/edit the photos in a collection. In the interest of keeping the project file fairly small, all the photos are loaded from the internet. Unfortunately, this means the pages can take a while to load, but on a real site this would be a straightforward fix. Basketball Stats Visualization Tool
Role: Developer, Designer Tools: Python, HTML, CSS, Javascript
<
>
This is a project I've been working on to visualize NBA statistics. It plots a player or team's statistics over the course of a season. I got the box scores for every game last season using the NBA stats API (further endpoint required) and then reformatted the JSON responses in order to visualize the data the way I wanted. Right now you can only see the data from last season (2014-2015), but I plan to add more seasons in the future as well as playoff data. I also plan to add features to allow filtering of the data and to use normalized values. You can play around with the tool yourself or see the code.
